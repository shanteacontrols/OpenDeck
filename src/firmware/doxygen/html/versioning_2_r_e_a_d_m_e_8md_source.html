1 # Software versioning for AVR
2 
3 This module consists of two parts:
4 
5 1) C executable responsible for writing software version into compiled .BIN file (major, minor and revision point), as well as writing 7-byte hash and CRC of compiled binary.
6 
7 2) AVR module used to read written sofware version, hash and CRC from running application.
8 
9 ## C executable
10 
11 In order for this utility to work correctly, five files are needed:
12 
13 - `FLASH.bin`
14 - `MAJOR`
15 - `MINOR`
16 - `REVISION`
17 - `HASH`
18 
19 All files must be placed in `bin` directory.
20 
21 `MAJOR`, `MINOR` and `REVISION` files should contain single number representing software version point (max value of 65535).
22 `HASH` file should contain 7-byte long hash, such as:
23 
24 `01b5b2f`
25 
26 To calculate location at which version and hash are going to be written, the following formula is used:
27 `FLASH_SIZE - 6 - 7 - 2`
28 
29 `FLASH_SIZE` is determined by checking file size of supplied `FLASH.bin` file.
30 After that, 6 bytes are subtracted since there are three version points, each taking two bytes.
31 After that, 7 bytes are subtracted since hash contains 7 bytes.
32 Finally, two bytes are subtracted, since CRC requires two bytes.
33 
34 After software version and hash have been written to binary, application calculates CRC of entire binary and writes it to last two memory addresses.
35 
36 ## AVR module
37 
38 To use AVR module for reading out software version and hash from running application, several symbols need to be supplied.
39 
40 ### `FLASH_SIZE`
41 
42 Size of running application in bytes. For instance, if flash size is 32kb, use the following symbol:
43 
44 - `FLASH_SIZE=32768`
45 
46 ### `EEPROM_SIZE`
47 
48 Size of EEPROM on AVR in bytes. For instance, if EEPROM size is 1kb, use the following symbol:
49 
50 - `EEPROM_SIZE=1024`
51 
52 Module reads software version and hash directly from running binary. Module can also detect if firmware has been updated by comparing CRC written in
53 running binary with CRC written to specified EEPROM address. If two differ, `checkNewRevision` function will return true, or false otherwise.
